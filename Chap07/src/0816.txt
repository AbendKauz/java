	- 부모 메소드 호출
		- 자식 클래스 내부에서 재정의된 부모 클래스 메소드를 호출해야 하는 경우
			명시적으로 super 키워드를 붙여 부모 메소드 호출
		ex) super.부모메소드();
	
	- final 키워드
		- 해당 선언이 최종 상태이며 결코 수정될 수 없음을 의미
		- 클래스 및 메소드 선언 시 final 키워드를 사용하면 상속과 관련됨
		
	- 상속할 수 없는 final 클래스
		- 부모 클래스가 될 수 없어 자식 클래스를 만들 수 없음을 의미
		ex) public final class 클래스{...}
		
	- 재정의할 수 없는 final 메소드
		- 부모 클래스에 선언된 final 메소드는 자식 클래스에서 재정의 할 수 없음
		ex) public final 리턴타입 메소드([매개변수, ...]){...}
		
- 자동타입변환(promotion)
	- 프로그램 실행 도중 자동으로 타입 변환이 일어나는 것
	[모양]
	부모타입 변수 = 자식타입;
       └─────┬─────┘
         자동타입변환
         
	ex) Cat cat = new Cat();┐
                            ├── Animal animal = new Cat();도 가능
		Animal animal = cat;┘
	- 바로 위 부모가 아니더라도 상속 계층에서 상위 타입인 경우 자동타입변환이 일어날 수 있음
	- 부모 타입으로 자동타입변환이 된 이후에는 부모 클래스에 선언된 필드 및 메소드만 접근 가능
	- 메소드가 자식 클래스에서 재정의 될 경우 자식 클래스의 메소드가 대신 호출됨
