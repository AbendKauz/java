모듈(Module)?
 - 외부에서 재사용 할 수 있는 패키지들을 묶은 것
 - 이클립스의 프로젝트는 하나의 모듈을 개발하는 것
 
모듈 기술자(Module-info.java)
 - 해당 모듈의 이름이 무엇인지,
 - 해당 모듈이 의존하는 모듈이 무엇인지
 (requires java.se;	-> 프로젝트 생성 시 항상 입력할 것)
 
모듈화가 필요한 이유
 - 패키지 보안
 - 작은 최적의 런타임(프로그램이 실행되는 시간) 을 만들기 위해
 
주석
 - 프로그램 실행과는 상관없이 코드에 설명을 붙여놓은 것
 - 복잡한 코드일수록 주석을 달면 전체 코드를 이해하기 쉽고 수정이 용이함
 * 라인주석 : ctrl + / 해제 : 동일
 * 범위주석 : ctrl + shift + /
 * 범위주석 해제 : ctrl + shift + \
 * 단축키 목록 : ctrl + l
 
실행문
 - main() 메소드 블록 내부에는 다양한 실행문들이 작성
 - 실행문 끝에는 반드시 세미콜론을 붙여서 실행문이 끝났음을 표시
